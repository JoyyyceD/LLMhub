# 社区评分帖规范（V1）

## 1. 功能定位

- 提供模型使用反馈与讨论能力
- 作为推荐旁路信号展示，不参与 V1 排序

## 2. 发帖入口与绑定规则

- 发帖入口有两个：
- 模型详情页入口
- 社区发帖入口
- 两个入口统一产出“模型评分帖”
- 发帖前必须绑定模型（需先选择 `model_id`，建议下拉选择）

## 3. 评分帖字段规则

- 必填：
- 总评分 `rating_overall`（1-5 星）
- 可选：
- 维度评分（中文维度）：`质量`、`价格`、`延迟`、`吞吐`、`稳定性`
- 服务商（provider_name，枚举）
- 做得好的地方（pros）
- 做得不好的地方（cons）
- 自由评论（comment）
- 发布规则：发布即展示（V1 不做审核流）

## 4. 唯一性与修改规则

- 约束：每用户每模型仅一条评分帖
- 同一用户对同一模型再次提交时，执行“更新”而非新增

## 5. 展示阈值

- 当评分数 `< 10`：不展示公开均分
- 当评分数 `>= 10`：展示均分与总评分数
- 评分帖列表始终可展示，不受 10 条阈值限制

## 6. 互动能力（V1）

- 用户可对某条评分帖发表评论（回复）
- 用户可对某条评分帖进行点赞或踩
- 同一用户对同一帖子仅保留一个态度（赞/踩二选一，可切换）

字段长度建议（V1）：

- `pros <= 200`
- `cons <= 200`
- `comment <= 800`
- `reply.content <= 300`

## 7. 社区列表排序与筛选

- 默认排序：按 `updated_at` 倒序
- 支持按模型筛选（`model_id`）

## 8. 与推荐关系

- V1：社区内容仅展示
- V1 不将社区分数写入推荐打分公式
