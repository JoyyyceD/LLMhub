# Contributing

## Scope

Contributions are welcome for:

- Methodology clarity
- API schema improvements (non-breaking)
- Example quality and docs readability
- Skill trigger/output consistency

## Pull request checklist

1. Keep changes backward compatible.
2. Add or update examples when request/response fields change.
3. Keep private logic out of public files.
4. Explain why the change improves recommendation transparency or usability.

## Non-goals

- Submitting private routing strategies
- Adding provider-specific private performance data
